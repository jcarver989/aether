The headless CLI runs a single prompt and exits — ideal for scripting, CI pipelines, and automation.

```bash
aether headless "Explain the main function in this project"
```

## Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `[PROMPT]...` | positional | — | Prompt text (joined with spaces) |
| `-m, --model` | `string` | — | Model spec (e.g., `anthropic:claude-sonnet-4-5`) |
| `-a, --agent` | `string` | — | Named agent from settings.json |
| `-C, --cwd` | `path` | `.` | Working directory |
| `--mcp-config` | `path` | — | Path to MCP config file |
| `--system-prompt` | `string` | — | Additional system prompt text |
| `--output` | `string` | `text` | Output format: `text`, `pretty`, `json` |
| `-v, --verbose` | flag | — | Verbose logging to stderr |

:::note
`--model` and `--agent` are mutually exclusive. If neither is set, Aether uses the first `userInvocable` agent from settings.json.
:::

## Output formats

<Tabs>
  <TabItem label="text">
    Plain text output — just the agent's response.

    ```bash
    aether headless --output text "What does this project do?"
    ```
  </TabItem>
  <TabItem label="pretty">
    Formatted output with markdown rendering.

    ```bash
    aether headless --output pretty "Summarize the README"
    ```
  </TabItem>
  <TabItem label="json">
    Structured JSON output — useful for piping into other tools.

    ```bash
    aether headless --output json "List all TODO comments" | jq '.response'
    ```
  </TabItem>
</Tabs>

## Reading from stdin

If no prompt is given and stdin is not a TTY, Aether reads the prompt from stdin:

```bash
echo "Explain this code" | aether headless
```

```bash
cat error.log | aether headless "What went wrong?"
```

This is useful for piping file contents or command output into Aether.

## Examples

### Quick question with a specific model

```bash
aether headless -m anthropic:claude-sonnet-4-5 "What does the Config struct do?"
```

### Using a named agent

```bash
aether headless -a researcher "Find all API endpoints in this project"
```

### CI pipeline

```bash
aether headless \
  -m anthropic:claude-haiku-4-5 \
  --output json \
  "Review the staged changes and list any issues" \
  > review.json
```

### Different working directory

```bash
aether headless -C /path/to/project "Summarize the architecture"
```

## Other subcommands

### show-prompt

Print the fully assembled system prompt (useful for debugging):

```bash
aether show-prompt
aether show-prompt --agent researcher
```

### init

Initialize a new Aether project:

```bash
aether init [path]
```
